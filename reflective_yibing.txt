2. I think one factor that is importance when evaluating the legality of web scraping is "whether the action of scraping avoids the website to provide its support to its intended users". For example, when Resultly crawled QVC's retail site, it actually sent too many requests that broke the website, leading to a 2 day crash, when no one can functionally use the website anymore, and leading to millions of loss. QVC is the victim of this web-scraping action since Resultly was dealing with the website in a way that it was not supposed to be dealt with. Otherwise, if the web-scraping action is controlled and it doesn't not affect the normal usage of website, it won't cause a large problem.

3. The first guideline is that if web-scraping is not used to provide welfare for those who provides data (otherwise, for example, is used to make profit) then it is not ethical to do so. The second guideline is that if web-scraping is down without the informed consent of both the data provider and the website being scrapped, then it is not ethical.
